# 预览海报版式对齐设计

## 目标

将首页、答题页和结果页的真实交互界面收敛到仓库现有预览海报的构图，
使移动端用户看到的标题、题目、选项、结果信息与主要行动入口具有与预览图
一致的视觉层级，同时保留现有答题、计分、结果分享和图片保存行为。

## 设计依据

以下素材是唯一的版式和视觉语言来源，均为 941 × 1672 PNG：

- `public/assets/home-poster-v3.png`
- `public/assets/quiz-poster-v3.png`
- `public/assets/result-poster-v3.png`

`share-poster-v3.png` 仅维持既有分享面板背景，不属于本轮逐屏重构范围。
现有 `src/styles.css` 已引用上述页面海报；本轮不替换、裁切或重新生成这些素材。

## 方案

页面海报继续承担静态装饰与构图锚点；可访问、可点击和随数据变化的内容仍由
DOM 渲染。每个交互区域采用半透明可读层而非不透明整块遮罩，让海报的标题、
红线、纹理和边角物件保持可见。

### 首页

- 保留海报内的主标题、宣言与边角装饰作为视觉主体。
- 将“开始鉴定”固定为首屏最显著、全宽的行动按钮，位置与海报红色按钮区域对齐。
- “先看看这是什么”维持为次级入口；说明抽屉不改变首页构图。
- 不再叠加第二套可见的标题、宣言或说明文字，避免与海报中文字重影。

### 答题页

- 以海报内的品牌、题号和进度线为基础，交互层只补充实时题号和真实进度宽度。
- 题目区域在窄屏中限制行数与字号扩张，优先保证五个答案卡片在首屏可达。
- 答案卡片使用低不透明旧纸层、细描边和红色选中态；选中时必须可感知，
  但不得遮住海报中的红线与线团。
- “上一题”保留在页面底部，与“不存在标准答案”的提示组成稳定的页脚。

### 结果页

- 保留海报主标题、人物拼贴、四轴面板和推荐阅读的顺序，动态内容只替换结果名称、
  分数、描述与推荐数据。
- 将“生成图卡”保留为结果主行动，紧邻首屏信息末端；“再测一次”为次级操作。
- 两段说明在窄屏维持并列卡片，字体和间距以不挤压标题与行动入口为约束。

## 响应式与可访问性

- 基准视口：375 × 812、390 × 844、430 × 932；桌面用 1280 × 900 验证居中电话画布。
- 页面主体最大宽度仍为 430px；背景图在移动端按画布宽度等比显示，不使用会拉伸比例的
  固定高度背景尺寸。
- 所有可点击元素保持原生 `button`，保留可见焦点、`aria-label` 与禁用态。
- `prefers-reduced-motion: reduce` 不新增持续动画。

## 非目标

- 不改题库、计分算法、结果数据、分享图卡导出尺寸或外链结构。
- 不创建新视觉资产，不引入第三方字体、图像服务或浏览器端密钥。
- 不创建或部署 Sites；发布必须在 UI 验收后单独获得确认。

## 验收标准

- 首屏不出现重复标题或重复宣言；主按钮与次级入口均可操作。
- 答题页在 375px 宽视口中，题目、五个选项与上一题入口没有裁切、重叠或横向滚动；
  点击任意选项后可见红色选中态并按现有流程前进。
- 结果页能显示动态结果名称、四轴分数、两段说明和“生成图卡”；点击后仍能打开分享面板。
- `npm test -- --run`、`npm run typecheck` 与 `npm run build` 通过。
